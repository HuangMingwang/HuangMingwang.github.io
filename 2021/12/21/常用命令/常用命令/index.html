vi/vim

Docker

退出dock容器: ctrl + D 或者输入exit

查看本地镜像: docker images

查看容器运行情况: docker ps

重启容器内的服务: docker restart nginx

查docker插件位置: docker volume inspect es-plugins

查看docker启动状态: systemctl status docker

启动docker服务: systemctl start docker

停止docker服务: systemctl stop docker

重启docker服务: systemctl restart docker

查看帮助: docker –help

加入开机自启: systemctl enable docker

显示所有容器包括停止的: docker ps -a

将多个镜像打包: docker -o xxx.tar 镜像1 镜像2

加载镜像: docker -i xxx.save.tar

MACOS

查ip：ifconfig | grep “inet “ | grep -v 127.0.0.1

显示隐藏文件夹: shift + command + .

切换桌面:control + 左和右和上和下

idea

跳出当前代码块：shift + command + enter

切换至上一个光标: option + command + 左右

重构: refactor

格式化代码: option + command + L

类和文件查询: shift + shift

全部内容查询: command + shift + R

当前文件查询: command + f

复制当前行: command + d

显示类的继承图: crtl + alt + U

调出idea寄存器保存的最近复制的N份内容,可选择性粘贴: shift + comamnd + v

补全当前行: shift+cmd + enter

将代码段封装成函数: option + command + m (按一次就默认生成,按两次为自定义生成)

for循环快捷方式: 100.for

快捷trycatch: option + command + T

if a == null : a.null

if (flag) : flag.if

while (flag): flag.while

Rename: shift+fn+F6

插件

  • 一键生成set方法的插件：generateallsetter option+enter

Typora

开发者工具: Shift + F12

清除格式: command + \

跳出列表或者引用: caps lock + enter

跳出代码块: command + enter

源代码模式: command + /

加入目录: [TOC]

文档树模式: control+ command +3

大纲模式: control + command + 1

Windows

切换虚拟桌面: win + Tab

新建虚拟桌面: win + ctrl + d

关闭虚拟桌面: win + ctrl + F4

显示桌面/返回当前页面: win + d

打开文件资源管理器: win + e

打开设置: win + i

锁屏: win + L

打开运行窗口: win + r

快捷打印: win + p

任务切换: alt + tap

浏览器内窗口切换: ctrl + tap

关闭当前窗口: alt + f4 / 双击窗口左上方

浏览器关闭当前活动窗口: ctrl + w

idea

快捷键 作用
Ctrl + Y 删除一行
Ctrl + D 复制一行
Ctrl + Alt + L 格式化
Ctrl + Alt + O 导包
Alt+Insert（可修改） New菜单\代码生成菜单（生成getter和setter，maven依赖等）
Ctrl + / 注释
Ctrl + Shift + / 多行注释
Ctrl + Alt + 左/右 回退到上一次操作的地方，等于eclipse中的 Alt+左/右
Shift + Alt + 上/下 将代码上移或下移一行

Ctrl + H ：罗列类的继承关系

连按两次shift，全局搜索

DOS

查看端口: netstat -ano

查看具体端口占用情况: netstat -aon|findstr “49157”

查进程名称: tasklist |findstr “进程id号”

查看会话: netstat -n

查看服务: msconfig

杀进程: tskill pid

杀进程: taskkill /f /t /im “进程id或者进程名称”
